Architecture

All services run behind a single nginx gateway on port 8080. Each container runs FastAPI (Python) serving both the REST API and a built React dashboard as static files.

Quick Start

Run everything with Docker

docker compose up --build

Then open http://localhost:8080 to see the gateway landing page with links to all services.

Run a single service without Docker

# Backend
cd Hotel/backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Frontend (separate terminal)
cd Hotel/frontend
npm install && npm run dev

API Key Authentication

Every API endpoint is protected by an API key. Provide it in one of two ways:

# Header (recommended)
curl -H "X-API-Key: hotel-1-key-abc123" http://localhost:8080/hotel-1/api/rooms

# Query parameter
curl "http://localhost:8080/hotel-1/api/rooms?api_key=hotel-1-key-abc123"

For LLM / AI Agents

Each service exposes a self-describing schema endpoint at GET /<service>/api/schema that returns a JSON manifest with all endpoints, parameters, payloads, and domain values.

For comprehensive offline documentation, download the Agent Reference file from each service page. These agent.md files contain everything an AI agent needs: authentication, workflows, request/response schemas, seed data, and error handling.

Adding More Instances

To add a 3rd hotel or 4th restaurant:

1. Add to docker-compose.yml

hotel-3:
  build:
    context: ./Hotel
    args:
      BASE_PATH: /hotel-3/
  environment:
    API_KEY: hotel-3-key-xyz789
  restart: unless-stopped

2. Add to gateway/nginx.conf

location /hotel-3/ {
    proxy_pass http://hotel-3:8000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}

3. Add to gateway's depends_on list, then rebuild

docker compose up --build
