Restaurant Reservation API

Table reservations with time slots, party size validation, and location-based seating

Download Agent Reference Open Dashboard

Overview

Simple restaurant reservation API designed for LLM / AI-agent consumption. Manages 12 tables across 4 locations (indoor, outdoor, patio, bar) with 6 fixed time slots per day. Each table can only be reserved once per time slot per day.

Quick Start

Backend (FastAPI)

cd Resturant/backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8001

Frontend (React + Vite)

cd Resturant/frontend
npm install
npm run dev

API Endpoints

GET /api/tables
List all tables with number, capacity, and location.
GET /api/tables/available
Find tables available for a specific date and time slot.
ParameterTypeRequired
datedate (YYYY-MM-DD)Yes
time_slotstring (HH:MM)Yes
party_sizeintegerNo
GET /api/time-slots
List all valid time slots.
POST /api/reservations
Create a new reservation.
Body FieldTypeRequired
table_idintegerYes
guest_namestringYes
guest_emailstringYes
datedate (YYYY-MM-DD)Yes
time_slotstring (HH:MM)Yes
party_sizeinteger (≥1)Yes
special_requestsstringNo
GET /api/reservations
List all reservations. Filter by date and/or status.
GET /api/reservations/{id}
Get details of a specific reservation.
DELETE /api/reservations/{id}
Cancel a reservation.
